# SeedBox

SeedBox is a public instrument lab built as one system across firmware,
simulator, desktop, hardware, tests, and docs.

It is not trying to hide its experimental character. The goal is to make that
character legible: one shared seed-driven core, several bodies, and a visible
proof culture.

## What SeedBox is

- Teensy 4.0 firmware
- laptop-native simulator
- JUCE standalone / VST3 lane
- deterministic golden-render test environment
- hardware build path
- teaching-oriented documentation environment

## Why it feels different

- The docs are part of the instrument, not an afterthought.
- The tests and golden renders act as public receipts.
- Hardware, simulator, and desktop builds are framed as related bodies of the
  same system.
- The repo keeps experimental edges visible instead of burying them in vague
  language.
